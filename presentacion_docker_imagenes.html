Anatomía de una Imagen

Una imagen es un conjunto de **capas de sólo lectura**. Al arrancar un contenedor, se añade una capa superior temporal de lectura/escritura.

Capa del Contenedor (Lectura/Escritura)
Capa 3: Archivos de la App (Inmutable)
Capa 2: Entornos de ejecución (Nginx/Python)
Capa 1: Distribución Base (Alpine/Ubuntu)
  • Las capas se comparten entre imágenes idénticas para **ahorrar espacio en disco**.
  • La inmutabilidad garantiza que el entorno siempre arranque exactamente igual sin importar la estación de trabajo.

1. Buscar y Descargar PROBAR ANTES

Localiza componentes en el registro público de Docker Hub antes de descargarlos localmente.

# Buscar imágenes oficiales filtrando por popularidad
docker search nginx --filter "is-official=true"
# Bajar la imagen al disco duro local sin iniciarla
docker pull ubuntu:22.04

💡 Si no especificas la versión (ej. `:22.04`), Docker descargará automáticamente la etiqueta `:latest`.

2. Listado e Inspección

Comprueba qué tienes almacenado localmente y audita los pasos de su creación:

# Listar imágenes disponibles en tu host local
docker images
# Descubrir el tamaño y comandos de cada capa interna
docker history nginx

3. Mantenimiento y Espacio

Es crucial mantener limpio el entorno para evitar saturar el almacenamiento durante las prácticas.

# Eliminar una imagen concreta (por TAG o ID)
docker rmi alpine:latest
# Liberar espacio purgando todo lo que esté sin uso
docker system prune -a

Prune borrará contenedores parados, redes no utilizadas e imágenes huérfanas de golpe.

4. Automatización con Dockerfile

Para crear soluciones reproducibles, declaramos los pasos en un archivo de texto plano llamado Dockerfile:


                        # 1. Elegimos la imagen base
                        FROM alpine:3.19

                        # 2. Instalamos dependencias
                        RUN apk update && apk add curl

                        # 3. Directorio interno
                        WORKDIR /app

                        # 4. Acción predeterminada
                        CMD ["curl", "https://ifconfig.me"]
  • FROM: Origen de la infraestructura.
  • RUN: Comandos ejecutados durante la compilación.
  • WORKDIR: Establece el contexto de ejecución.
  • CMD: Qué comando corre al arrancar el contenedor por defecto.

5. Compilación y Etiquetado

Transforma tu archivo descriptivo `Dockerfile` en una imagen empaquetada lista para funcionar.

# Construir la imagen indicando el directorio actual (.)
docker build -t mi-app:1.0 .
# Etiquetar con tu estructura de usuario para subir a la nube
docker tag mi-app:1.0 tu-usuario-hub/mi-app:1.0

6. Compartir y Desplegar

Registro Online

Ideal para entornos de red conectados:

docker login

docker push user/repo:tag

Archivo Físico (.tar)

Ideal para laboratorios aislados o pendrives:

docker save -o app.tar img

docker load -i app.tar
