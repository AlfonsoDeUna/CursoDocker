UD 06: Docker Compose

Orquestación de aplicaciones multi-contenedor

Curso de Introducción a Docker y Kubernetes

¿Qué es Docker Compose?

Es una herramienta para **definir y ejecutar** aplicaciones Docker de múltiples contenedores.

  • Usa un fichero **YAML** para la configuración.
  • Lanza todos los servicios con un solo comando.
  • Actúa como un orquestador simple a nivel local.
docker compose version

Nota: En versiones actuales se integra como plugin de Docker (sin guion).

El Formato YAML

"YAML Ain't Markup Language". Basado en sangrías (indentación).

# Estructura básica
servicios:
  base_datos:      # Objeto
    imagen: mysql  # Par clave/valor
    puertos:       # Lista
      - "3306:3306"
  • Pares clave/valor: `imagen: nginx`
  • Listas: Empiezan con un guion `-`.
  • Indentación: Fundamental para definir jerarquías (como en Python).

docker-compose.yml

Conceptos clave que podemos definir:

services: Los contenedores que forman la app.

image: Imagen base del servicio.

build: Directorio con un Dockerfile propio.

ports: Mapeo de puertos (Host:Contenedor).

depends_on: Orden de arranque de servicios.

environment: Variables de entorno.

volumes: Persistencia de datos compartida.

restart: Política de reinicio (always, on-failure).

Ejemplo: Wordpress + DB

version: "3.9"
services:
  db:
    image: mariadb:10.11
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: example
  wordpress:
    depends_on:
      - db
    image: wordpress:latest
    ports:
      - "8000:80"

Lanzamos con: docker compose up -d

Ciclo de Vida (Comandos)

up -d: Crea y arranca en segundo plano.

ps: Lista los contenedores del proyecto.

stop / start: Detiene o inicia servicios.

down: Para y **elimina** contenedores y redes.

logs -f: Ver la salida de todos los servicios.

Escalado Local

Docker Compose permite replicar servicios fácilmente:

docker compose up -d --scale app=3

Esto levantará 3 instancias del servicio 'app'. El balanceo de carga interno se gestiona mediante el DNS de Docker.
