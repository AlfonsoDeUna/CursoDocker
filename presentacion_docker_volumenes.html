Redes y Volúmenes

Comunicación y persistencia de datos en contenedores

Curso de Introducción a Docker y Kubernetes

Redes Predefinidas

  • Bridge: La red por defecto. Usa la interfaz docker0 del host.
  • Host: Elimina el aislamiento. El contenedor usa la IP y puertos del host directamente.
  • None: Aislamiento total. Solo loopback.
# Ver interfaces en el host
ip a show docker0

# Listar redes Docker
docker network ls

Crear Redes Propias

Crear redes personalizadas permite **aislamiento** y **resolución DNS automática** por nombre de contenedor.

# Crear una red bridge aislada
docker network create red-proyectos
# Conectar contenedores con nombres DNS
docker run -d --name db --network red-proyectos mariadb
docker run -it --network red-proyectos alpine ping db

¿Por qué necesitamos Volúmenes?

"Los contenedores son efímeros"

Si borras el contenedor, los datos internos se pierden.

Los volúmenes permiten que los datos sobrevivan al ciclo de vida del contenedor.

Estrategias de Persistencia

Bind Mounts

Mapeas una ruta absoluta del host.

Ideal para compartir código fuente en desarrollo.

Docker Volumes

Gestionados por Docker en /var/lib/docker/volumes.

La opción recomendada para producción.

* También existe tmpfs: almacenamiento veloz en RAM (no persistente).

Uso de Bind Mounts

Sincroniza una carpeta de tu ordenador con el contenedor:

# Usando -v (Ruta Host : Ruta Contenedor)
docker run -d -v /home/user/web:/var/www/html nginx
# Usando --mount (Más explícito y recomendado)
docker run -d --mount type=bind,source=/home/user/web,target=/usr/share/nginx/html nginx

Gestión de Volúmenes

Docker gestiona el almacenamiento por ti.

# Crear y listar volúmenes
docker volume create datos-db
docker volume ls
# Montar volumen en un contenedor
docker run -d -v datos-db:/var/lib/mysql mariadb

Copias de Seguridad

Para hacer un backup de un volumen, lanzamos un contenedor temporal que lo comprima:

# Backup del volumen 'mis-datos' a un .tar en el host
docker run --rm --volumes-from contenedor-activo \
-v $(pwd):/backup ubuntu \
tar cvf /backup/backup_enero.tar /ruta/datos/interna
Este patrón monta los volúmenes del contenedor origen en uno de backup para realizar la copia.
